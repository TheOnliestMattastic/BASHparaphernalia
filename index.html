üêß NEWbara.sh - System Bootstrap Automation

Purpose: Comprehensive Nobara/Fedora system bootstrapper with configuration-driven package management, automatic visual feedback, and safety-first design.

Key Features

  • Configuration-Driven Installation: Requires explicit package file input (‚àíd or ‚àíf) preventing accidental installations
  • External Package Lists: Load DNF packages and Flatpak apps from separate config files for reusability
  • Automatic Visual Dependencies: Auto-installs cowsay/lolcat early for guaranteed colorful feedback throughout execution
  • Dry-Run Safety: Preview installations without making changes; cleanup operations skipped during dry-runs
  • Multi-Package Manager: Seamless DNF and Flatpak integration with user/system installation modes
  • Timestamped Logging: All output logged for troubleshooting and audit trails
  • Smart Cleanup: Removes orphaned packages and unused Flatpak runtimes post-installation

Installation & Setup

cd BASHparaphernalia/scripts/NEWbara
chmod +x NEWbara.sh

# Install from configuration file (required)
./NEWbara.sh -d config/dnf-packages.txt

# Install from Flatpak apps configuration
./NEWbara.sh -f config/flatpak-packages.txt

# Install both with system-wide Flatpaks
./NEWbara.sh -d config/dnf-packages.txt -f config/flatpak-packages.txt -S

# Preview without installing (dry-run)
./NEWbara.sh -d config/dnf-packages.txt -n

Configuration Files

Create custom package lists in plain text format (one per line):

# config/my-dnf-packages.txt
neovim
git
python3

# config/my-flatpak-apps.txt
org.videolan.VLC
md.obsidian.Obsidian
org.kde.krita

Dependencies: dnf, flatpak, sudo (cowsay/lolcat auto-installed)

Note: cowsay and lolcat can be safely uninstalled after running: sudo dnf remove cowsay lolcat

üåà LOGme.sh - Terminal-Based Login Enhancement

Purpose: Sophisticated login enhancement using wezterm split-pane orchestration to display RGB setup, system visualization, fortune messages, and task logging concurrently.

Demo

Video shows: Autoload prompt ‚Üí User input (y) ‚Üí Wezterm split panes with cbonsai animation and fortune display ‚Üí MTrclone task execution

Key Features

  • Split-Pane Terminal Display: Left pane shows OpenRGB setup then cbonsai animation; right pane displays fortune with syntax highlighting then runs MTrclone
  • Wezterm Orchestration: Advanced terminal control for concurrent visual feedback
  • OpenRGB Integration: Automatic RGB profile loading for ambient lighting control
  • System Visualization: Bonsai tree animation provides engaging visual feedback during initialization
  • Fortune Display: Random quotes with cowsay ASCII art, syntax-highlighted via bat
  • Task Logging: Visible execution of additional startup scripts (MTrclone.sh)
  • Smart Autostart: Detects existing configuration; prompts for setup only when needed
  • Cross-Desktop: Works with both KDE Plasma and GNOME (.desktop file autostart)
  • Graceful Error Handling: Continues execution despite individual component failures

Installation & Setup

# Install dependencies (Fedora/Nobara)
sudo dnf install wezterm openrgb fortune cowsay bat cbonsai

# Make executable and run
cd BASHparaphernalia/scripts/LOGme
chmod +x LOGme.sh
./LOGme.sh

# Script will prompt to set up autostart if not already configured

Dependencies: wezterm, openrgb, fortune, cowsay, bat, cbonsai

Notes: Requires OpenRGB to be configured with a profile named "Home"; runs automatically at login via .desktop file in ~/.config/autostart/

üîÑ MOOcase.sh - Batch File/Directory Case Conversion

Purpose: Standardizes file and directory naming by batch converting first character case with flexible filtering and optional visual feedback.

Key Features

  • Case Conversion Modes: Lowercase by default; uppercase with ‚àíu flag
  • Selective Processing: Process directories only (‚àíd), files only (‚àíf), or both
  • Custom Path Targeting: Specify target directory with ‚àíp flag (defaults to current directory)
  • Safe Renaming: Only renames items when new name differs from original
  • Visual Feedback: Cowsay/lolcat messages when available (graceful fallback)
  • Comprehensive Logging: Detailed output of all rename operations
  • Error Handling: Robust recovery with clear error messages

Installation & Setup

# No dependencies required (optional cowsay/lolcat for visual feedback)
cd BASHparaphernalia/scripts/MOOcase
chmod +x MOOcase.sh

# Convert files to lowercase in current directory
./MOOcase.sh

# Convert directories only to uppercase
./MOOcase.sh -u -d

# Convert files in specific directory to lowercase
./MOOcase.sh -p /path/to/directory -f

Dependencies: None required (optional: cowsay, lolcat for enhanced feedback)

üöÄ Getting Started

Prerequisites

  • Linux system (tested on Nobara/Fedora; adaptable to other distributions)
  • Bash shell (standard on most Linux distributions)
  • Git (for cloning the repository)
  • Sudo access (for system-level operations)

# Or make individual scripts executable
chmod +x scripts/NEWbara/NEWbara.sh

# ... etc

Run Individual Scripts

# NEWbara system bootstrap
./scripts/NEWbara/NEWbara.sh -d scripts/NEWbara/config/dnf-packages.txt

# LOGme login enhancement
./scripts/LOGme/LOGme.sh

# MTrclone cloud mounts
./scripts/MTrclone/MTrclone.sh

Script-Specific Requirements

  • NEWbara.sh: Requires dnf, flatpak, sudo access, and at least one package configuration file (‚àíd or ‚àíf)
  • LOGme.sh: Requires wezterm, openrgb, fortune, cowsay, bat, cbonsai
  • MTrclone.sh: Requires rclone configured with cloud account remotes
  • DONtype.sh: Requires xdotool and X11 display server
  • MOOcase.sh: No dependencies (optional cowsay/lolcat for visual feedback)

Note: Each script includes help documentation: run with --help or -h flags for detailed options.
